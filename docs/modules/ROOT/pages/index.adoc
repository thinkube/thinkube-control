*thinkube-control* is the platform's control plane — the component that deploys
your apps, manages models and GPUs, builds images, and runs the services that the
rest of Thinkube depends on.

== Two faces, one control plane

thinkube-control is a FastAPI backend with a Vue web dashboard. Almost everything
it can do is exposed two ways:

* *As MCP tools* — the way you actually drive the platform. From code-server you
  ask Claude, and the request runs against the control plane (deploy a template,
  load a model, check GPU status, build a venv, mirror a model…). This is the
  signature Thinkube workflow.
* *As the web control panel* — the same capabilities with a visual dashboard, for
  when you want to see and click rather than ask.

== What it manages

* *Templates & deployments* — deploy and redeploy app templates onto the cluster.
* *Models & GPUs* — load, serve, and mirror LLMs; inspect GPU status.
* *Images & registries* — build custom images and manage Harbor.
* *Notebooks & venvs* — JupyterHub virtual environments and notebooks.
* *Components & services* — install and toggle the platform's services.

thinkube-control is a *core component*: it ships with the platform rather than as a
user-deployed app.
